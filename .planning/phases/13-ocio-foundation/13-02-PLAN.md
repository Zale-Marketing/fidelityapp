  <files>app/api/ocio/config/route.ts</files>
  <action>
    Creare app/api/ocio/config/route.ts con due handler:

    **GET:** Legge la riga ocio_config del merchant autenticato.
    - Auth: header Authorization: Bearer {token} → supabase.auth.getUser(token). Return 401 se non auth.
    - Plan check: query merchants.plan per il merchant_id del profilo, return 403 se plan !== 'business'.
    - Query: SELECT * FROM ocio_config WHERE merchant_id = $merchantId LIMIT 1.
    - Se non esiste ancora la riga, return { data: null } (la pagina settings inizializza i default lato client).
    - Return 200 con { data: OcioConfig | null }.

    **PATCH:** Upsert della riga ocio_config per il merchant autenticato.
    - Auth + plan check identici al GET (401/403 se fallisce).
    - Body accettato: { google_maps_url?, module_reviews?, module_alerts?, alert_whatsapp_number?, alert_min_rating? }
    - Validazione: se google_maps_url presente e non vuota, deve iniziare con 'https://' — altrimenti 400 con { error: 'URL non valido' }.
    - Operazione: supabase.from('ocio_config').upsert({ merchant_id, ...body, updated_at: new Date().toISOString() }, { onConflict: 'merchant_id' }).
    - Return 200 con { success: true }.

    Usare createClient con SUPABASE_SERVICE_ROLE_KEY (non il client browser). Non usare edge runtime — lasciare Node runtime default.
  </action>
