1. PATCH /api/ocio/config con { google_maps_url, module_reviews, module_alerts }
2. Se ok: setSaved(true)

Modifica da apportare — aggiungere DOPO il PATCH /api/ocio/config riuscito:
```typescript
// Gestione schedule Trigger.dev — non-blocking
const scheduleAction = config.module_reviews ? 'create' : 'cancel'
try {
  await fetch('/api/ocio/schedule', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${accessToken}`,
    },
    body: JSON.stringify({ action: scheduleAction }),
  })
  // Ignora errori schedule — non blocca il salvataggio config
} catch {
  // Silenzioso — Trigger.dev potrebbe non essere configurato in dev
}
```

IMPORTANTE: la chiamata schedule e fire-and-forget rispetto all'UX del salvataggio. L'utente vede "Salvato!" indipendentemente dal successo della chiamata schedule. Solo errori critici di config (dal PATCH /api/ocio/config) mostrano il messaggio di errore rosso.

Variabili gia disponibili in saveConfig:
- `accessToken` — token Bearer gia estratto in loadData()
- `config.module_reviews` — stato del toggle
- `mapsUrl` — URL Google Maps (gia validato nel PATCH precedente)

La logica da aggiungere:
- Se `config.module_reviews === true` (toggle attivo): chiama con `action: 'create'`
- Se `config.module_reviews === false` (toggle disattivo): chiama con `action: 'cancel'`
- Usa `fetch('/api/ocio/schedule', { method: 'POST', headers: { 'Content-Type': 'application/json', 'Authorization': `Bearer ${accessToken}` }, body: JSON.stringify({ action: scheduleAction }) })`
- Wrap in try/catch — silenzioso in caso di errore (non mostrare all'utente, non bloccare il flusso)
- NON await in modo che blocchi il feedback "Salvato!" — await e accettabile ma deve essere dentro il blocco "if (res.ok)" dopo setSaved(true)

Il flow completo di saveConfig dopo la modifica:
```
1. setError(null), setSaving(true)
2. PATCH /api/ocio/config
3. Se !res.ok: setError, setSaving(false), return
4. setSaved(true), setTimeout setSaved(false)
5. POST /api/ocio/schedule (fire-and-forget, silent catch)
6. setSaving(false)
```

NON modificare nient'altro nella pagina — solo la funzione saveConfig. Non aggiungere nuovi import, non modificare il JSX, non cambiare altri handler.

Assicurarsi che la modifica sia TypeScript-valida (no any impliciti, accessToken e gia string nel scope).
