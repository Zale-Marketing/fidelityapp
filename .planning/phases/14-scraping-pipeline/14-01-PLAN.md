Crea il file `trigger/ocio-scraper.ts` con un task Trigger.dev che:

1. **Struttura base:** Usa `schedules.task({ id: "ocio-review-scraper", cron: "0 */6 * * *", maxDuration: 300, run: async (payload) => { ... } })`. Il cron e un placeholder (lo schedule creato dinamicamente via API route usa il cron reale) — mettere cron comunque per compatibilita Trigger.dev.

2. **Logic del run:**
   - Crea un client Supabase con service role key
   - Legge `ocio_config` per TUTTI i merchant con `module_reviews = true` e `google_maps_url IS NOT NULL`:
     ```
     supabase.from('ocio_config').select('merchant_id, google_maps_url').eq('module_reviews', true).not('google_maps_url', 'is', null)
     ```
   - Per ogni merchant trovato, esegue `scrapeForMerchant(merchantId, googleMapsUrl, supabase)`
   - Logga il totale: `logger.log("ocio-scraper complete", { totalMerchants: N })`

3. **Funzione `scrapeForMerchant`:**
   - Conta le recensioni esistenti per il merchant: `supabase.from('ocio_reviews').select('id', { count: 'exact', head: true }).eq('merchant_id', merchantId)`
   - `maxReviews = (count ?? 0) === 0 ? 50 : 20`
   - Crea `ApifyClient({ token: process.env.APIFY_TOKEN })`
   - Chiama l'actor: `client.actor('compass/google-maps-reviews-scraper').call({ startUrls: [{ url: googleMapsUrl }], reviewsSort: 'newest', maxReviews }, { waitForFinishSecs: 120 })`
   - Se `run.status !== 'SUCCEEDED'`: `throw new Error(`Apify run failed: ${run.status}`)` — Trigger.dev gestira il retry
   - Legge items: `client.dataset(run.defaultDatasetId).listItems()`
   - Mappa ogni item in un oggetto `ocio_reviews`:
     - `review_id`: `${item.reviewerId ?? 'anon'}_${item.publishedAtDate ?? item.publishAt ?? 'nodate'}` — usa i campi disponibili in modo difensivo
     - `merchant_id`: merchantId
     - `author_name`: `item.name ?? null`
     - `author_url`: `item.reviewerUrl ?? null`
     - `rating`: `item.stars ?? item.rating ?? null`
     - `text`: `item.text ?? item.textTranslated ?? null`
     - `published_at`: `item.publishedAtDate ?? item.publishAt ?? null`
     - `owner_reply`: `item.responseFromOwnerText ?? null`
     - `owner_reply_at`: `item.responseFromOwnerDate ?? null`
     - `review_url`: `item.reviewUrl ?? null`
     - `place_id`: `item.placeId ?? null`
     - Tutti i campi `ai_*`: null (li popola Phase 15)
     - `ai_is_fake`: false
     - `reply_status`: `'pending'`
     - `alert_sent`: false
   - Filtra items con `review_id` valido (no undefined/null risultato dal mapping)
   - Fa upsert batch: `supabase.from('ocio_reviews').upsert(reviews, { onConflict: 'merchant_id,review_id', ignoreDuplicates: true })`
   - Aggiorna `ocio_config`: `supabase.from('ocio_config').update({ last_scrape_at: new Date().toISOString() }).eq('merchant_id', merchantId)`
   - Logga: `logger.log("merchant scraped", { merchantId, fetched: items.length, upserted: reviews.length })`

4. **Gestione edge case:**
   - Se `items` e vuoto o undefined: logga `logger.log("no reviews found", { merchantId })` e ritorna senza errore
   - Recensioni senza testo (solo rating): mappare normalmente — `text` sara null, e valido per `ocio_reviews`
   - Wrap tutta la logica per-merchant in try/catch: se un merchant fallisce, loga l'errore e continua con il successivo (non bloccare l'intera run per un merchant)

Crea `app/api/ocio/schedule/route.ts` — API route Node runtime (NON edge) che gestisce la creazione e cancellazione degli schedule Trigger.dev per il merchant autenticato.

**Auth pattern:** Copia identicamente la funzione `getAuthenticatedMerchant` da `app/api/ocio/config/route.ts` — stessa logica: Bearer token → getUser → profiles → merchants.plan === 'business'.

**POST handler — logica:**
```
POST /api/ocio/schedule
Body: { action: 'create' | 'cancel' }
```

1. Autentica il merchant (BUSINESS plan)
2. Legge il body JSON, estrae `action`
3. Se `action === 'create'`:
   - Legge `google_maps_url` da `ocio_config` per il merchant
   - Se `google_maps_url` e null/empty: return `{ error: 'URL Google Maps non configurato' }` 400
   - Crea/aggiorna lo schedule:
     ```typescript
     const schedule = await schedules.create({
       task: "ocio-review-scraper",
       cron: "0 */6 * * *",
       externalId: merchantId,
     })
     ```
   - Salva `trigger_schedule_id` in `ocio_config`:
     ```typescript
     await supabase.from('ocio_config').update({ trigger_schedule_id: schedule.id }).eq('merchant_id', merchantId)
     ```
   - Return `{ success: true, scheduleId: schedule.id }`
4. Se `action === 'cancel'`:
   - Legge `trigger_schedule_id` da `ocio_config`
   - Se non presente: return `{ success: true }` (niente da cancellare)
   - Cancella con `schedules.del(scheduleId)` — wrap in try/catch (lo schedule potrebbe gia non esistere in Trigger.dev)
   - Aggiorna `ocio_config`: `{ trigger_schedule_id: null }`
   - Return `{ success: true }`
5. Se `action` non riconosciuta: return `{ error: 'Azione non valida' }` 400

**Import necessari:**

**Nota importante:** `schedules.create` di Trigger.dev v3 e idempotente su `externalId` — se viene chiamato due volte con lo stesso `externalId`, aggiorna lo schedule esistente invece di crearne uno nuovo. Questo copre il caso "cambio URL" (non serve recreate, il prossimo run legge l'URL aggiornato da DB).

**Error handling:** Ogni operazione Trigger.dev in try/catch. Se `schedules.create` fallisce (es. TRIGGER_SECRET_KEY non configurato in dev): return 500 con `{ error: 'Errore Trigger.dev: ' + err.message }`.
