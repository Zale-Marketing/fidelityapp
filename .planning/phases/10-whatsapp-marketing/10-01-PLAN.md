Create directory structure `app/api/whatsapp/connect/` and `app/api/whatsapp/status/`.

**app/api/whatsapp/connect/route.ts** — POST handler:
1. Get user from Supabase auth, look up profile → merchant_id (use SUPABASE_SERVICE_ROLE_KEY).
2. Return 401 if not authenticated.
3. Check if merchant already has maytapi_phone_id set — if yes, return `{ phoneId: existing_phone_id }` without creating a new one (idempotent).
4. Call Maytapi `POST /addPhone` with empty body `{}`.
5. Extract phone_id from response — check both `data.id` and `data.pid` (open question from research — use `addData?.id ?? addData?.pid ?? addData?.data?.id ?? addData?.data?.pid`).
6. If phone_id is missing from response, log the full response body and return 502.
7. Update merchants table: `{ maytapi_phone_id: phoneId, maytapi_session_status: 'pending' }`.
8. Return `{ phoneId }`.

Error handling: Catch fetch errors, return 500 with message.

**app/api/whatsapp/status/route.ts** — GET handler with `?action=` param:
1. Get user from Supabase auth, look up profile → merchant_id (use SUPABASE_SERVICE_ROLE_KEY).
2. Return 401 if not authenticated.
3. Look up merchant — get maytapi_phone_id, maytapi_session_status, maytapi_daily_count, maytapi_last_reset_date.
4. If no phone_id, return `{ status: 'not_connected', dailyCount: 0, dailyLimit: 200 }`.
5. Parse `?action` from URL searchParams.

**action=qr:**
- Call Maytapi `GET /{phoneId}/qrCode`.
- Proxy binary PNG response: `return new Response(buffer, { headers: { 'Content-Type': 'image/png', 'Cache-Control': 'no-store' } })`.
- If Maytapi returns non-200, return a 404 text/plain response.

**action=status (default):**
- Call Maytapi `GET /{phoneId}/status`.
- Parse `statusData.data.status` (values: 'active' | 'qr-code' | 'idle' | 'phone-error').
- If status is now 'active' and DB has a different value, update `merchants.maytapi_session_status = 'active'`.
- Compute today's date as `new Date().toISOString().split('T')[0]`.
- Daily count: if `maytapi_last_reset_date !== today`, effective count is 0; else use `maytapi_daily_count`.
- Return `{ status, dailyCount, dailyLimit: 200 }`.
