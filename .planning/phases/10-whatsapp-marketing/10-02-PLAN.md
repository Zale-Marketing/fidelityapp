Phone normalization function (from research):
```typescript
function normalizePhone(phone: string): string | null {
  const cleaned = phone.replace(/[\s\-\(\)\.]/g, '')
  if (cleaned.startsWith('39')) return cleaned
  if (cleaned.startsWith('+39')) return cleaned.slice(1)
  if (cleaned.startsWith('0039')) return cleaned.slice(2)
  if (cleaned.match(/^[3][0-9]{9}$/)) return `39${cleaned}`
  return null
}
```

Create `app/api/whatsapp/send/route.ts` as a POST handler. NOT edge runtime — standard Node.js runtime.

**Request body:**
```typescript
{
  recipients: Array<{ phone: string; name?: string }>
  message: string
}
```

**Handler logic:**

1. Auth check: get user → profile → merchant_id using SUPABASE_SERVICE_ROLE_KEY. Return 401 if missing.
2. Parse body — validate `recipients` is array, `message` is non-empty string. Return 400 on validation failure.
3. Load merchant: query `merchants` for `id, maytapi_phone_id, maytapi_session_status, maytapi_daily_count, maytapi_last_reset_date`.
4. Check `maytapi_phone_id` — if null/empty, return 400 `{ error: 'WhatsApp non connesso. Collega il tuo numero nelle impostazioni.' }`.
5. Check session status — if not 'active', return 400 `{ error: 'Sessione WhatsApp non attiva. Riconnetti il tuo numero.' }`.
6. **Rate limit check:**
   - `const today = new Date().toISOString().split('T')[0]`
   - `const needsReset = merchant.maytapi_last_reset_date !== today`
   - `const currentCount = needsReset ? 0 : (merchant.maytapi_daily_count ?? 0)`
   - If `currentCount >= 200` → return 429 `{ error: 'Limite giornaliero di 200 messaggi raggiunto. Riprova domani.' }`
7. **Phone normalization:** Apply `normalizePhone()` to each recipient.phone. Skip recipients where normalizePhone returns null (track in `skippedCount`).
8. **Rate limit capacity:** After normalization, check that `currentCount + validRecipients.length <= 200`. If over limit, truncate list to `200 - currentCount` recipients and note in response.
9. **Batch send (batches of 10, 200ms delay between batches):**
   ```
   const MAYTAPI_BASE = `https://api.maytapi.com/api/${process.env.MAYTAPI_PRODUCT_ID}`
   for each batch of 10:
     await Promise.allSettled(batch.map(r => callMaytapi(phone_id/sendMessage, { to_number, type:'text', message })))
     await new Promise(resolve => setTimeout(resolve, 200))
   ```
   Track `successCount` and `failCount`.
10. **Update daily counter:**
    ```
    await supabase.from('merchants').update({
      maytapi_daily_count: currentCount + successCount,
      maytapi_last_reset_date: today,
    }).eq('id', merchantId)
    ```
11. Return `{ sent: successCount, failed: failCount, skipped: skippedCount, dailyCount: currentCount + successCount, dailyLimit: 200 }`.

Include helper function `normalizePhone(phone: string): string | null` and `callMaytapi(path, options)` in the same file.

Error handling: Wrap everything in try/catch, return 500 on unexpected errors.
