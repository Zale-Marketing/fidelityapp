<objective>
Add basic request authentication to the /api/wallet and /api/wallet-update routes so that arbitrary external callers cannot trigger Google Wallet API calls with arbitrary cardIds.

Purpose: Both routes currently accept any POST request with a cardId — no auth check. An external attacker could enumerate cardIds and trigger unlimited wallet API calls, consuming Google Wallet API quota and revealing card data. The fix uses a shared secret in an internal API header, validated server-side.
Output: Both wallet API routes reject requests missing the correct Authorization header with 401, while the internal callers (stamp page, card page) continue working.
</objective>

From app/c/[token]/page.tsx (calls wallet):
This file calls POST /api/wallet with { cardId } — must add the Authorization header.

The auth strategy: use a shared internal secret stored in env var INTERNAL_API_SECRET.
Callers send: `Authorization: Bearer ${process.env.NEXT_PUBLIC_INTERNAL_API_SECRET}`
Routes verify: header matches `process.env.INTERNAL_API_SECRET`

Using NEXT_PUBLIC_ prefix for the caller side because callers are client components.
This is a lightweight anti-abuse measure — it prevents external enumeration while keeping implementation simple (no user session required since wallet actions are triggered client-side).

    In BOTH `app/api/wallet/route.ts` and `app/api/wallet-update/route.ts`, add the following auth check at the very start of the POST handler, before any other logic:

    ```typescript
    // Basic auth check — reject external callers
    const authHeader = request.headers.get('authorization')
    const expectedSecret = process.env.INTERNAL_API_SECRET
    if (expectedSecret && authHeader !== `Bearer ${expectedSecret}`) {
      return NextResponse.json({ error: 'Non autorizzato' }, { status: 401 })
    }
    ```

    The guard is `if (expectedSecret && ...)` — when the env var is not set (e.g., in local dev without the variable configured), the check is skipped and the route behaves as before. This prevents breaking existing development environments while securing production.
