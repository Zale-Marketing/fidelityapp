<task type="auto">
  <name>Task 1: Fix idempotency key in app/stamp/page.tsx</name>
  <files>app/stamp/page.tsx</files>
  <action>
    Add a new state variable `const [idempotencyKey, setIdempotencyKey] = useState('')` at the top of the component alongside the other state variables.

    In the `processCode` function, immediately after extracting the token and before calling any handler, generate the key:
    ```typescript
    const scanKey = `${cardId}-${crypto.randomUUID()}`
    setIdempotencyKey(scanKey)
    ```
    Where `cardId` is the card.id retrieved from the database lookup. Generate it AFTER the card is found but BEFORE the program-type branching logic runs.

    Then update ALL transaction insert calls to use the scan-time key instead of Date.now(). The functions that contain `idempotency_key` inserts are:
    - `addStamp` — change `idempotency_key: \`${card.id}-${Date.now()}\`` to `idempotency_key: idempotencyKey || \`${card.id}-${Date.now()}\``
    - `addPoints` — same pattern
    - `addCashback` — same pattern
    - `addTierSpend` — same pattern
    - `useSubscription` — same pattern
    - `activateSubscription` — same pattern
    - `redeemCashback` — same pattern
    - `redeemStampsReward` — same pattern
    - `redeemPointsReward` — same pattern

    Also reset `idempotencyKey` to `''` in the `resetScanner` function so a fresh key is generated for the next scan.

    Note: `crypto.randomUUID()` is available in modern browsers and Node.js 14.17+. No import needed. This ensures that even if the cashier confirms the same scan twice (e.g., double-tap on the confirm button), the second transaction insert will be rejected by the database's unique constraint on `idempotency_key` — preventing duplicate stamps.
